# GeoAgent Solution Workspace 详细设计稿

中文名：GIS 解决方案智能工作台  
产品简称：GeoAgent 工作台  
设计方向：Solution Studio  
标语：From Requirements to GIS Deliverables / 从客户需求到方案交付

## 1. 设计目标

这套设计以“企业级 GIS AI 方案交付工作台”为核心，不再做偏演示型的大屏，而是做适合售前、方案、交付团队长期使用的 SaaS 工作台。

核心感受：

- 登录页保留强 GIS 科技感，突出 3D 地球、Agent 节点、需求到交付链路。
- 系统页采用高级浅色企业工作台，提升信息密度、可读性和长期办公舒适度。
- 左侧导航使用深墨绿，建立 GIS 专业感和平台识别度。
- 平台总览、项目详情、流程编排、架构预览局部保留深色沉浸画布，用于三维模型、空间分析、Agent 编排等核心能力展示。

## 2. 视觉系统

### 2.1 颜色

| 类型 | 色值 | 用法 |
| --- | --- | --- |
| 页面背景 | `#F4F7F6` | 系统页主背景 |
| 卡片背景 | `#FFFFFF` | 表格、表单、统计模块 |
| 主品牌色 | `#167D64` | 主按钮、选中态、进度、重点图标 |
| 空间数据色 | `#0891B2` | 地图路径、三维场景高亮、数据连线 |
| 主文字 | `#18211E` | 标题、核心数字 |
| 次级文字 | `#667570` | 描述、辅助信息 |
| 边框 | `#DFE7E3` | 表格线、输入框、模块分隔 |
| 告警色 | `#F59E0B` | 待处理、风险、警告 |
| 错误色 | `#DC5A5A` | 失败、异常、错误 |
| 深色导航 | `#14211D` | 左侧导航、沉浸画布底色 |

### 2.2 字体和尺寸

- 标题：`20-24px / 600`，用于页面标题和核心模块标题。
- 二级标题：`16-18px / 600`，用于卡片标题、分区标题。
- 正文：`14px / 400`，用于表格、描述、表单。
- 辅助文字：`12-13px / 400`，用于状态、时间、说明。
- 数字指标：`28-36px / 600`，用于平台总览关键统计。

### 2.3 圆角和阴影

- 主卡片圆角：`8px`。
- 输入框、按钮圆角：`6px`。
- 列表行不做独立卡片，使用轻分隔线。
- 阴影只用于浮层、弹窗、侧边抽屉，不做满屏卡片堆叠。

## 4. 登录页设计

### 4.1 页面结构

左侧 65%：深色 GIS 科技视觉。  
右侧 35%：白色登录面板。

左侧内容：

- 品牌：GeoAgent Solution Workspace
- 中文名：GIS 解决方案智能工作台
- 标语：AI-powered GIS solution delivery
- 大型 3D 地球，以亚洲/中国区域为默认视角。
- 地球周围显示四个核心节点：需求洞察、方案生成、架构设计、PPT 交付。
- 背景显示星网、地图栅格、Agent 连接线、微弱城市灯光。

右侧登录面板：

- 标题：登录工作台
- 用户名、密码、记住我、忘记密码
- 主按钮：登录工作台
- 次操作：使用企业 SSO 登录

### 4.2 登录页动效

推荐使用 Three.js + Anime.js 组合：

- 3D 地球缓慢自转，周期 `36s`，线性无限循环。
- Agent 节点沿轨道轻微浮动，周期 `3.2s`，错峰 `delay`。
- 节点之间的连线出现流光粒子，周期 `2.4s`。
- 四个交付阶段节点按顺序点亮：需求洞察 -> 方案生成 -> 架构设计 -> PPT 交付。
- 登录面板进入时从右侧 `24px` 位移淡入，持续 `520ms`。
- 输入框聚焦时边框变为品牌绿，背景轻微提亮。
- 登录按钮 hover 时出现从左到右的微弱光扫。

### 4.3 动效克制原则

登录页可以更强动效，但不能影响表单操作。所有循环动效都应低对比、慢节奏，鼠标进入表单区时降低背景动效亮度。

## 5. 平台总览

### 5.1 页面目标

平台总览是进入系统后的“运营驾驶舱”，回答三个问题：

- 当前有多少项目在推进？
- Agent 交付链路是否健康？
- 哪些任务、风险、交付物需要我处理？

### 5.2 推荐布局

第一行：欢迎区 + 主操作。

- 标题：欢迎回来，张伟
- 副标题：高效组织项目交付，让每一次方案都更有价值。
- 主按钮：新建项目
- 次按钮：运行 Flow

第二行：统计区，建议 6 个指标，排版为 `4 + 2` 或 `6 等分`。

关键统计：

- 进行中项目
- 本月交付项目
- 待办任务
- 方案复用率
- Agent 成功率
- 平均交付周期

统计卡片设计：

- 每张卡片左侧图标，右侧主数字。
- 数字下方显示环比变化。
- 不使用大面积彩色背景，只使用品牌绿/青色/橙色的小面积状态点。
- 复用率、成功率建议使用小型环形进度或 sparkline。

第三行：核心工作区。

左侧 8 栅格：项目进展表格。

- 项目名称
- 客户
- 当前阶段
- 状态
- 负责人
- 进度
- 更新时间

右侧 4 栅格：AI 助理推荐。

- 项目风险提醒
- 案例推荐
- 知识推荐

第四行：交付全链路概览。

- 横向节点：需求分析、产品匹配、案例检索、架构设计、方案撰写、PPT 生成。
- 每个节点显示运行中、已完成、待开始数量。
- 当前活跃节点用品牌绿实心态，待开始用浅灰。

### 5.3 平台总览动效

- 统计数字进入时使用 count-up，持续 `800ms`。
- 进度条从 0 到目标值，持续 `700ms`。
- 交付链路节点按顺序淡入，间隔 `80ms`。
- AI 推荐卡片 hover 时右侧出现轻微展开箭头。

## 17. 动效系统

### 17.1 动效分级

一级动效：品牌和核心能力展示。

- 登录页 3D 地球。
- 项目详情三维架构预览。
- 流程编排运行态。

二级动效：工作台反馈。

- 数字递增。
- 进度条加载。
- 表格行 hover。
- 节点状态切换。

三级动效：微交互。

- 按钮 hover。
- 输入聚焦。
- 抽屉出现。
- 提示淡入。

### 17.2 Anime.js 建议

适合用 Anime.js 实现：

- 登录页节点依次点亮。
- 统计数字 count-up。
- 进度条和环形进度。
- Flow 连线流光。
- 卡片进入和列表 stagger。
- 按钮光扫和 hover 位移。

### 17.3 Three.js / R3F 建议

适合用 Three.js / React Three Fiber 实现：

- 登录页地球。
- 项目详情三维园区/城市模型预览。
- 架构能力热点。
- 空间图层切换。

## 18. 前端落地顺序

建议按以下顺序改造：

1. 全局主题、导航、布局、中文乱码修复。
2. 登录页视觉和动效。
3. 平台总览统计排版。
4. 项目作战室和项目详情。
5. 流程编排和架构预览。
6. Agent 技能、方案模板、运行日志。
7. 用户权限、IMA、模型、GitHub 配置。

这样能先建立整体设计语言，再逐步补齐业务页面。
